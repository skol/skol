Мое знакомство с ним приключилось, когда я познакомился с Ruby on Rails и его языком ruby. Тема была интересная, но, на тот момент, ни как не востребованная, в наших краях. Стал искать что - то подобное. И нашел:
## Django
Я не первый, и далеко не последний, кого этот фреймворк привел в разработку вообще, и в web разработку на python в частности.
Коротенько, все что нужно, на мой взгляд, помнить/знать об этом фреймворке я собрал [здесь](django/README.md).
## Flask
Не помню причин знакомства с ним. Минималистичный, быстрый, как минимум, по сравнению с django, надежный, понятный. Самое большое достоинство - гибкость. Ни какие ограничения на разработчика не накладываются, нужно тебе данные в редисе хранить, вместо БД, пожалуйста, используй нужный коннектор и пользуйся, не нужен html, не подключай jinja, нужно что - то делать дополнительно, помимо обработки запроса, middleware к твоим услугам. Количество батареек под разные задачи вряд ли уступит django. Склерозник по этому фреймворку [здесь](flask/README.md).
