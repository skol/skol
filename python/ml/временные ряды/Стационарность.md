# Стационарность
Стационарность — это **свойство**. **Не любой ряд можно сделать стационарным** (или хотя бы не любой — с практической пользой).

Когда мы говорим "ряд стационарен", мы утверждаем, что его **статистические характеристики не зависят от времени**:
- **Математическое ожидание** постоянно
- **Дисперсия** постоянна
- **Автоковариация** зависит только от лага, а не от момента времени

Это **свойство** ряда, как "красный" или "тяжелый". Ряд либо обладает им, либо нет.
# Преобразования — это инструменты
Преобразования (разности, логарифм, Box-Cox, деление на скользящее стандартное отклонение) — это способы **приблизить** ряд к стационарности.

Но:
- **Не всегда получается**
- **Не всегда нужно**
- **Не всегда имеет смысл**

# Когда ряд НЕЛЬЗЯ сделать стационарным
## Случай 1: Детерминированный тренд + детерминированная сезонность
```python
y = 0.5 * t + sin(2π * t / 12) + ε
```
Такие ряды можно сделать стационарными, **если знать структуру**:
- Вычесть тренд (по МНК)    
- Вычесть сезонные индексы

Но если тренд или сезонность меняют форму — становится сложно.
## Случай 2: Ряд с изменяющейся структурой (structural breaks)
```python
# До 1980 года: AR(1) с μ=0
# После 1980 года: AR(1) с μ=50
```
Такие ряды невозможно сделать стационарными одним глобальным преобразованием.
## Случай 3: Интегрированные процессы высшего порядка
```python
# I(2) — нужно дважды брать разность
y_t = 2*y_{t-1} - y_{t-2} + ε_t
```
Технически можно взять diff(2), но:
- Увеличивается дисперсия ошибок    
- Потеря данных на первых двух наблюдениях    
- Сложнее интерпретировать модель
## Случай 4: Стохастическая сезонность с изменяющимся периодом
Если сезонный паттерн меняется (например, праздники с плавающей датой), фиксированная сезонная разность может не сработать.
## Случай 5: Гетероскедастичность, которая не лечится логарифмом
Если дисперсия меняется не пропорционально уровню, а скачкообразно или по сложному закону, простые преобразования могут не помочь.
# А как же тест Дики-Фуллера?
Тест ADF проверяет гипотезу:
- **H₀:** ряд имеет единичный корень (нестационарен)    
- **H₁:** ряд стационарен

Если p-value < 0.05, мы _отвергаем_ H₀ и считаем ряд стационарным.
Но:
- Если p-value = 0.04, ряд "стационарен" только на уровне значимости 5%    
- Если p-value = 0.10, формально не стационарен, но может быть достаточно стационарен для моделирования    
- **Тест не говорит, можно ли преобразовать ряд** — только оценивает текущее состояние
# Практический подход
В реальной работе мы не гоняемся за "идеальной стационарностью". Мы ищем **достаточную стационарность**:

| Состояние                                                | Что делаем                                                             |
| -------------------------------------------------------- | ---------------------------------------------------------------------- |
| Явный тренд + сезонность                                 | diff(1) + diff(season) — почти всегда работает                         |
| Дисперсия растет                                         | log или Box-Cox                                                        |
| После преобразований ACF/PACF выглядят "хорошо"          | Останавливаемся                                                        |
| После преобразований ADF p-value < 0.05                  | Останавливаемся                                                        |
| После разумных преобразований ADF p-value все еще > 0.05 | **Останавливаемся**, если ряд визуально стационарен (иногда тест врет) |
# 6. Примеры из практики
**Можно сделать стационарным (легко):**
- Продажи с трендом и сезонностью → log + diff(1) + diff(12)    
- Температура воздуха (циклическая, без тренда) → уже стационарна    
- Курс валют с трендом → diff(1) достаточно
**Сложно или невозможно:**
- Индекс фондового рынка за 100 лет (меняющаяся волатильность, структурные сдвиги)
- Популяция животных с вымиранием (тренд + изменение сезонности)
- Данные с накоплением (cumulative sums) — там I(2) или выше

**Резюме:** стационарность — это свойство, к которому мы _стремимся_ через преобразования, но не всегда достигаем идеала. На практике "достаточно стационарно" часто бывает достаточно.

# Почему создается впечатление, что ряд можно "замучить до стационарности"
Потому что в учебных примерах:
1. Данные **специально подобраны** — там гарантированно есть решение    
2. Преобразований немного (diff, log, diff(season))    
3. Всегда показывают успешный кейс

Но если бы в примере был ряд, который **нельзя** сделать стационарным, студенты бы запутались. Поэтому показывают только "хорошие" ряды.
