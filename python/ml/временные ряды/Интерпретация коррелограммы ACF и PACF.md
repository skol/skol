# Что показывает ACF (Autocorrelation Function)
**ACF** — корреляция ряда с его же лагами, **включая цепные эффекты**.

Пример: корреляция с лагом 3 может быть высокой не потому, что t и t-3 связаны напрямую, а потому что t связан с t-1, t-1 с t-2, t-2 с t-3.

**Как читать ACF:**
- Каждый бар — это корреляция с соответствующим лагом
- Синяя область — доверительный интервал (обычно 95%)
- Если бар **выходит за синюю границу** — корреляция статистически значима
# Что показывает PACF (Partial Autocorrelation Function)
**PACF** — корреляция ряда с лагом **после удаления влияния промежуточных лагов**.

Пример: PACF на лаге 3 покажет прямую корреляцию между t и t-3, исключив влияние t-1 и t-2.
## Типичные паттерны и их интерпретация

| Паттерн ACF                                               | Паттерн PACF                 | Что это значит                                                               |
| --------------------------------------------------------- | ---------------------------- | ---------------------------------------------------------------------------- |
| Медленно затухает (несколько лагов)                       | Резкий обрыв после 1-2 лагов | **AR процесс** (авторегрессия). Порядок AR = последний значимый лаг PACF     |
| Резкий обрыв после q лагов                                | Медленно затухает            | **MA процесс** (скользящее среднее). Порядок MA = последний значимый лаг ACF |
| Оба медленно затухают                                     | Оба медленно затухают        | **ARMA / ARIMA** — смешанный процесс                                         |
| Пики на определенных лагах (7, 14, 24, 30, 365 и кратных) | Пики на тех же лагах         | **Сезонность**. Период = лаг первого значимого пика (кроме лага 1)           |
## Примеры для наглядности
**Пример 1: AR(1) процесс**
- ACF: медленно экспоненциально затухает (хвост)
- PACF: резкий обрыв после лага 1 (только первый бар значим)

**Пример 2: MA(1) процесс**
- ACF: резкий обрыв после лага 1
- PACF: медленно затухает

**Пример 3: Нестационарный ряд (случайное блуждание)**
- ACF: затухает **очень медленно**, почти линейно
- PACF: первый бар ~0.9, остальные могут быть значимы, но мелкие

**Пример 4: Сезонный ряд (без тренда)**
- ACF: пики на 12, 24, 36 лагах (если годовая сезонность)
- PACF: пики на тех же лагах (если SAR процесс)
# Как это использовать для приведения к стационарности
1. **Смотрим на исходный график** — есть тренд? дисперсия растет?
2. **Смотрим на ACF** — если медленно затухает (хвост), нужна разность  (обычно 1)
3. **Берем разность** (или логарифм + разность)
4. **Смотрим на ACF/PACF после разности**:
	- Если ACF резко обрывается, PACF медленно затухает → MA компонента
    - Если PACF резко обрывается, ACF медленно затухает → AR компонента
    - Если есть пики на кратных лагах → нужна сезонная разность
### Простой алгоритм для первого домашнего задания задания
Для каждого из 6 рядов:
1. **Визуальный осмотр** графика — тренд? сезонность? растущая дисперсия?
2. **Если дисперсия растет** → `log` или `Box-Cox`
3. **ACF исходного ряда** — если медленно затухает → нужна разность
4. **Взять разность** (обычно `diff(1)`)
5. **Посмотреть на ACF/PACF после разности**:
	- Если есть значимые пики на кратных лагах → взять сезонную разность `diff(season)`
6. **Проверить стационарность**:
	- Визуально (график, ACF/PACF)
    - Тест Дики-Фуллера (p-value < 0.05 — стационарен)
### Что должно получиться в итоге
Стационарный ряд — это когда:
- Нет тренда (значения колеблются вокруг нуля)
- Дисперсия примерно постоянна
- ACF быстро затухает (нет медленного хвоста)
- Нет значимых пиков на сезонных лагах
