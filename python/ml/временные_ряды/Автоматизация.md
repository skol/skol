**Существуют** довольно много инструментов автоматической оптимизации параметров, но с важной оговоркой: в отличие от GridSearchCV в sklearn, здесь **нельзя просто так перебрать все комбинации**, потому что:
1. **Порядок преобразований имеет значение**    
2. **Не все комбинации имеют смысл**    
3. **Оценка "качества" стационарности** — это не одна метрика

## Определение порядка дифференцирования (d)

**Тест KPSS** (Kwiatkowski-Phillips-Schmidt-Shin) — определяет, сколько раз нужно взять разность:
```python
from statsmodels.tsa.stattools import kpss

# KPSS: H0 — ряд стационарен
kpss_stat, p_value, lags, crit = kpss(df['Count'])
```

Можно построить автоматический подбор:
```python
def find_d(series, max_d=3):
    for d in range(max_d + 1):
        if d == 0:
            test_series = series
        else:
            test_series = series.diff(d).dropna()
        
        _, p_value, _, _ = kpss(test_series)
        if p_value > 0.05:  # стационарен по KPSS
            return d
    return max_d
```

# Полноценные "GridSearch" для временных рядов

## pmdarima (auto_arima)

Это самый популярный инструмент. Он перебирает параметры (p, d, q, P, D, Q, S) и выбирает лучшие по AIC/BIC:
```python
import pmdarima as pm

# Автоматический подбор SARIMA
model = pm.auto_arima(
    df['Count'],
    seasonal=True,
    m=12,  # период сезонности
    start_p=0, start_q=0, max_p=3, max_q=3,
    start_P=0, start_Q=0, max_P=2, max_Q=2,
    d=None,  # автоопределение
    D=None,  # автоопределение сезонной разности
    trace=True,
    error_action='ignore',
    stepwise=True
)

print(model.summary())
```
**Что умеет auto_arima:**
- Подбирает d и D (порядки дифференцирования)    
- Перебирает p, q, P, Q    
- Учитывает сезонность    
- Выбирает по AIC/BIC    

**Чего не умеет:**
- Сам применять Box-Cox (но можно передать transformed)    
- Учитывать порядок преобразований (это моя ответственность)

## Кастомный "GridSearch" для преобразований
Если хочется самому перебрать варианты:
```python
import numpy as np
from statsmodels.tsa.stattools import adfuller
from scipy.stats import boxcox

def evaluate_stationarity(series):
    """Оценка стационарности по ADF"""
    result = adfuller(series.dropna())
    return {
        'adf_stat': result[0],
        'p_value': result[1],
        'is_stationary': result[1] < 0.05
    }

# GridSearch по преобразованиям
transformations = [
    ('original', lambda x: x),
    ('log', lambda x: np.log(x)),
    ('boxcox', lambda x: boxcox(x)[0] if (x > 0).all() else x),
    ('diff1', lambda x: x.diff()),
    ('log_diff1', lambda x: np.log(x).diff()),
    ('diff12', lambda x: x.diff(12)),
    ('log_diff1_diff12', lambda x: np.log(x).diff().diff(12)),
]

results = []
for name, transform in transformations:
    try:
        transformed = transform(df['Count'])
        stats = evaluate_stationarity(transformed)
        stats['name'] = name
        stats['n_obs'] = len(transformed.dropna())
        results.append(stats)
    except Exception as e:
        print(f"{name} failed: {e}")

# Сортируем по p-value (чем меньше, тем лучше)
results_df = pd.DataFrame(results)
print(results_df.sort_values('p_value'))
```
# Почему это не всегда нужно
В учебных задачах **ручной подход** часто предпочтительнее:
1. **Учишься читать графики и ACF** — это навык, который потом пригодится    
2. **Понимаешь физический смысл** — почему именно diff(1) и diff(12)    
3. **Не попадаешь в ловушку "подогнал под тест, но смысла нет"**    

Автоматические методы хороши, когда:
- Есть сотни рядов (например, по каждому магазину/складу)    
- Нужно быстрое решение    
- Есть бюджет на валидацию
